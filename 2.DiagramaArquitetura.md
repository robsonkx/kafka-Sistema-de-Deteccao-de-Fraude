### 1 Diagrama de Arquitetura

A arquitetura é baseada em **processamento de eventos em tempo real**, com separação entre três caminhos principais de processamento:

- **Hot Path** → decisão imediata (\<1s)
- **Warm Path** → processamento analítico e features (minutos)
- **Cold Path** → armazenamento histórico e treinamento de ML

#### 1️⃣ Camada de Ingestão

Responsável por receber as transações.

Componentes:

- **POS (Maquininhas)**

  - Origem das transações
  - Comunicação segura via **mTLS**
- **API Gateway**

  - Controle de acesso
  - Rate limit
  - Autenticação
- **Load Balancer (HAProxy / NGINX)**

  - Balanceamento entre instâncias
  - Health checks
  - Active-active


#### 2️⃣ Camada de Streaming

O **Kafka Cluster** é o backbone do sistema.

Características:

- 72 partitions
- replication factor \= 3
- retenção \= 7 dias


Principais tópicos:

-*transactions.raw
-transactions.validated
-fraud.alerts
-fraud.confirmed*


#### 3️⃣ Hot Path — Detecção Imediata

Objetivo: **responder em menos de 1 segundo**

Componentes:

- **Fraud Check Service (Go / Rust)** 
  Microserviço que avalia cada transação.

*fraud:cards
fraud:users
fraud:sites*

Latência esperada:

\< 5 ms


Fluxo:

transaction → fraud check → decisão


Saídas possíveis:

*fraud.alert
transaction.approved
transaction.rejected*


#### 4️⃣ Warm Path — Feature Engineering

Responsável por enriquecer dados.

Tecnologias:

- **Flink / Spark Streaming**

Funções:

- agregações em tempo real
- criação de features
- detecção comportamental


Exemplos de features:

*transações últimos 5 minutos
valor médio por usuário
distância geográfica
velocidade de uso do cartão*


As features são armazenadas em:

*Feature Store (Feast / Tecton)* 


#### 5️⃣ Cold Path — Histórico e ML

Armazenamento de longo prazo.

Tecnologia:

**Data Lake**

S3 / GCS

Conteúdo:

- raw logs
- datasets de ML
- histórico de 7 anos

Além disso:

**ML Inference**

TensorFlow Serving

Utilizado para scoring assíncrono ou batch.


#### 6️⃣ Camada de Persistência

Dados operacionais são persistidos em três bancos diferentes.

##### Cassandra

Uso:

transações


Estratégia:

partition key = site_id + mês
TTL = 180 dias


##### PostgreSQL

Uso:

dados mestre


Exemplos:

- usuários
- estabelecimentos
- contratos


##### Neo4j

Uso:

detecção de redes de fraude


Permite identificar:

- anéis de fraude
- centralidade
- conexões entre cartões e usuários
