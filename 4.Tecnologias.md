# - Tecnologias -

| Camada              | Tecnologia         | Motivo                         |
| --------------------- | -------------------- | -------------------------------- |
| Streaming           | Kafka              | alto throughput e resiliência |
| Cache               | Redis              | consultas em milissegundos     |
| Streaming analytics | Flink / Spark      | processamento distribuído     |
| Banco transacional  | Cassandra          | escalabilidade horizontal      |
| Banco relacional    | PostgreSQL         | consistência e integridade    |
| Banco de grafos     | Neo4j              | análise de redes de fraude    |
| Data Lake           | S3 / GCS           | armazenamento massivo          |
| Feature Store       | Feast / Tecton     | consistência de features      |
| ML Serving          | TensorFlow Serving | inferência escalável         |


# Kafka

O Apache Kafka é uma plataforma de streaming de eventos distribuída, de alta performance e tolerante a falhas, ideal para processar grandes volumes de dados em tempo real.

Kafka desacopla:

- ingestão
- detecção
- analytics
- armazenamento


# Redis

Lookup extremamente rápido


# Flink / Spark

Responsável por enriquecer dados.

Apache Flink e Spark são motores de processamento de Big Data de alta performance. O **Flink** brilha em streaming de dados em tempo real com baixíssima latência e gerenciamento complexo de estado. O **Spark** destaca-se pelo processamento em memória, sendo versátil para batch e stream, com alto desempenho para ETL e SQL

# Cassandra

Motivo:

- alta escrita
- escalabilidade horizontal
- baixa latência


# PostgreSQL

Motivo:

- consistência
- joins complexos


# Neo4j

Banco de dados orientado a grafos com eficiência em lidar com dados altamente conectados e relacionamentos complexos.


# S3 / GCS

O Amazon S3 e o Google Cloud Storage (GCS) oferecem armazenamento de objetos com durabilidade extrema (99,999999999%), alta disponibilidade, segurança robusta e escalabilidade infinita. Ambos reduzem custos com camadas de armazenamento automático (Intelligent-Tiering), facilitam a gestão de dados (ciclo de vida) e são ideais para grandes volumes de dados


# Feast / Tecton

**Feast**

- **Baixo Custo Inicial**: Sendo código aberto, não há custos de licenciamento.


**Tecton**

- **Pipelines de Transformação Totalmente Gerenciados**: Automatiza a criação de pipelines de dados em tempo real e em lote.


Tecton e Feast são as principais ferramentas de **Feature Store** no mercado de Machine Learning (ML), sendo o **Feast** a versão de código aberto e o **Tecton** a plataforma comercial gerenciada pelos mesmos criadores.


# TensorFlow Serving

O **TensorFlow Serving** é um sistema de alta performance projetado especificamente para colocar modelos de machine learning em produção com baixa latência e alto rendimento.
